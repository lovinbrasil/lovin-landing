# Texto Visível Para Usuários

Nunca coloque termos técnicos, nomes de infraestrutura ou detalhes internos na interface do usuário.
Quem usa o app e o painel pode ser uma pessoa leiga, então todo texto visível deve ser simples,
humano e orientado à ação.

Evite totalmente textos visíveis como Supabase, raw_app_meta_data, role, RLS, API, payload,
bucket, banco de dados, tabela, migration, token, stack trace, log, endpoint, query ou qualquer
outro detalhe técnico. Quando algo técnico precisar ser comunicado, traduza para linguagem de
produto. Exemplo: em vez de "Use uma conta Supabase com raw_app_meta_data.role como admin",
use "Use uma conta autorizada para acessar o painel".

Palavras com cedilha ou tio ou acentos tome cuidado para não escrever sem então

ç ou ã tome muito cuidado, outras também.
